Processes Tab

Click Processes to see the Processes you can use to analyze wave data. They are organised into groups and sub-groups to make them easier to locate.

  • Click a Process Group Name (for example, Web Processing Services from CSIRO O&A) to see the Web Process servers in that group. The list of process servers displayed is extracted from the metadata catalogue so there will be a slight delay whilst they are retrieved. When the process servers are displayed open Australian Wave Atlas PyWPS Server

  • Click a process Definition Name (for example Monthly Power Distribution Matrix) to specify the input parameters for this process and begin a run.

  • Click on the button of any Process Definition to show the available metadata for that Process. The metadata includes a description of the process and the input parameters that can be specified.

  • Fill out the inputs by clicking on the tabs and entering the values. Mandatory/Required input parameters have an asterisk (*) next to the name of the parameter.

  • Parameters that require a spatial input (eg. LonLatPosition) have a cross-hair button that can be used to obtain a position by clicking on the map.

  • Click on the Submit tab when the mandatory inputs have been specified.

  • When the OK button is clicked, the process will be submitted to the web process server. The process result will appear as a sub-item of the process definition.

  • Whilst the process is executing a spinner will be shown. Clicking on the spinner will allow cancellation of the process.

  • When the process finishes, the result will be show with the process name and the time of execution. In addition two icons will be shown to the right. The icons show the status of the process run and when clicked provide some operations:

    • - clicking on this icon will delete the process result from the process tab.
    • - indicates that the process has executed successfully. Clicking on this icon or the process result name will bring up a window with the results of the process. If appears instead then the process execution failed and the result of the process will be the error message returned by the process server.
